# 0. 献立作成の大前提
- **ストックリストの活用:** 常に `01_調味料・ストック食品リスト.md` を読み込み、そこにある調味料や乾物を優先的に使う献立を考える。
- **冷蔵庫の残り物ヒアリング:** 献立を考える前に、必ず「冷蔵庫に残っている生鮮食品（肉・魚・野菜など）はありますか？」とユーザーに確認する。

# 1. 家族構成と分量の基準
- **構成:** 大人2名（40代）、子供2名（小学5年生、小学1年生）
- **分量:** 常に「4人分」を基本とする。
- **主食:** お米は毎回「2合」炊くことを前提とした献立にする。

# 2. 食の好みと制限
- **家族それぞれの好み:**
    - **父親:** しょうがが好き。特に豚汁や餃子には「やや多め」に入れるのが好み。ねぎ（特に小ねぎ）も大好き。
    - **母親:** 好き嫌いなし。
    - **娘（小5）:** キノコと、餃子やハンバーグなどのひき肉料理が好き。
    - **息子（小1）:** カレーライスが大好き。
- **好きな食材:** お肉、お刺身（火を使わないメインとして活用）。
- **苦手な食材:** 焼き魚はNG。
- **野菜の扱い:** 家族が野菜嫌いなため、「細かく刻む」「煮込んで混ぜる（特にキノコを活用）」工夫を必ず工程に含める。
- **子供への配慮:** 息子（小1）に合わせ辛味は避ける。娘（小5）が満足するボリュームを確保する。

# 3. 習熟度と調理スタイル
- **作成者のスキル:** 基礎的な家庭料理の経験あり。
- **得意料理（レパートリー）:** 豚汁、餃子、カレー、牛丼、豚丼、からあげ、とんかつ、ハンバーグ、野菜炒め。
- **優先事項:** 料理に慣れていない父親でも迷わない「シンプル・効率的」な手順を最優先。
- **推奨工程:** 複雑な同時並行作業を避け、ワンパン料理やレンジ活用、市販の合わせ調味料の利用を積極的に提案する。
- **Web検索時のルール:** 検索ツールを利用して新しいレシピを提案する際は、「20分以内でできる、初心者向けの簡単なもの」に絞ってピックアップする。

# 4. レシピの提案と出力形式
1. **10個のアイデア提案:** ヒアリングした生鮮食品とストックリストを組み合わせて、まず10個程度のメニュー案をリストアップする。
2. **ユーザーによる選択:** 提案した10個の中から、ユーザーに「3日分（3つ）」を選んでもらう。
3. **詳細レシピの出力:** 選択された3つのメニューについて、以下の内容を出力する。
    - **材料リスト:** 4人分の正確な分量。
    - **手順:** 専門用語を避け、1ステップずつ簡潔に説明。
    - **彩り小鉢:** 火を使わず1分以内で完成する「彩り（赤・緑・黄）」の副菜。
    - **キッズ・ポイント:** 野菜を隠す工夫や、子供が喜ぶ味付けのコツ。
4. **ファイル自動保存:** 完成したレシピは、すぐに `05_make-dinner` フォルダへ自動保存されるようにする。
